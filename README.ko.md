## 문제

Claude Code는 강력합니다. 하지만 혼자 쓰면 코드를 한 번에 작성하고 끝입니다 — 리뷰 없이, 감사 없이, 품질 게이트 없이. 동작은 하지만 취약합니다.

Agent Harmony는 빠진 것을 채웁니다: **개발 프로세스 전체**.

| | 패스 | 리뷰 | 품질 |
|---|---|---|---|
| Claude Code 단독 | 1 | 없음 | 동작하지만 취약 |
| Claude Code + 직접 리뷰 | 3-5 | 수동 | 프로덕션급 |
| **Claude Code + Harmony** | **5+** | **자동** | **프로덕션급, 핸즈프리** |

## 빠른 시작

```bash
/plugin marketplace add hohre12/jwbae-plugins
/plugin install agent-harmony@jwbae-plugins

# 공식 플러그인 명령어
/agent-harmony:harmony 실시간 채팅 앱 — 채팅방, 인증, 파일 공유

# 플러그인 설치 후 많은 환경에서 동작하는 shorthand
/harmony 실시간 채팅 앱 — 채팅방, 인증, 파일 공유
```

공개 배포 경로는 이것으로 통일합니다: 마켓플레이스 추가 → 플러그인 설치 → Harmony 실행.

## 엔터를 누르면 일어나는 일

```
Phase 1 — 인터뷰                          객관식 질문 몇 개에 답하세요
Phase 2 — PRD 생성                        종합적인 스펙, 자동 생성
Phase 3 — 셋업 (자동)                     에이전트 팀 + 태스크 + 참조 문서
Phase 4 — 빌드 (완전 자동)                태스크마다:
                                            설계 → 구현 → 자기 리뷰
                                            → 품질 게이트 → 프로덕션 감사
                                            → 통과할 때까지 수정
Phase 5 — 완료                            프로덕션 레디 프로젝트
```

**가장 많이 개입하는 구간은 Phase 1입니다.** 그 이후에는 Harmony가 파이프라인을 계속 진행하고, 명시적 수동 인수 또는 중단 판단이 필요할 때만 다시 돌아옵니다.

## 하네스 엔지니어링

프롬프트 엔지니어링은 부탁합니다. **하네스 엔지니어링은 자동화가 가장 높은 품질에 도달할 가능성을 최대화합니다.**

파이프라인의 모든 단계가 제약입니다 — 제안이 아닙니다.

```
 1. PRD 생성
    명확한 요구사항 없이는 빌드할 수 없습니다.
    인터뷰에서 생성된 600줄 이상의 스펙 — 모호한 브리프가 아닙니다.

 2. 동적 에이전트 팀 (프로젝트 한정, 글로벌 공유 안 됨)
    당신의 프로젝트 기술 스택과 도메인에 맞는 전문 에이전트 생성.
    FastAPI 프로젝트와 Next.js 프로젝트는 다른 에이전트를 받습니다.
    에이전트는 해당 프로젝트에만 존재 — 다른 프로젝트와 오염 없음.

 3. 전문가 참조 문서 (할루시네이션 방지)
    각 에이전트가 검증된 도메인 지식 문서를 받습니다.
    학습 데이터 기억이 아닌 팩트 기반으로 작업합니다.

 4. 책임감 압박
    모든 에이전트가 압니다: 다른 에이전트가 당신의 작업을
    아무 맥락 없이 냉정하게 평가할 것이라는 사실을. 첫 품질이 달라집니다.

  5. 서버 교차 검증
     빌드, 테스트, 린트, 커버리지 — 측정 가능한 항목은 작업 중인 에이전트 바깥에서 교차 검증합니다.
     측정 가능한 품질 신호를 조용히 얼버무릴 수 없게 설계했습니다.

  6. 무한 품질 루프 (사용자 승인 기반 수동 인수 체크포인트 포함)
     감사 실패? → 수정 → 재감사. 반복.
     반복 실패 시 Harmony가 멈추고 계속 진행 / 수동 인수 / 중단을 선택하게 합니다.
```

Harmony는 기본적으로 자동 재시도를 계속합니다. 다만 비용과 시간을 고려해, 일정 횟수 실패 후 사용자가 직접 계속 진행 / 수동 인수 / 중단을 선택할 수 있습니다.

수동 인수는 Harmony의 자동 검증 루프를 종료하고, 이후 결과 책임을 사용자가 직접 집니다.

### 품질 임계값

코드로 강제됩니다. 프롬프트로 제안되는 게 아닙니다.

| 메트릭 | 기준 | 측정 방법 |
|--------|------|----------|
| 빌드 | 통과 필수 | 실제 빌드 명령어 |
| 테스트 | 통과 필수 | 전체 테스트 스위트 |
| 린트 | 에러 0개 | 프로젝트 린터 |
| 커버리지 | >= 70% | pytest --cov / jest --coverage |
| 파일 크기 | <= 400줄 | wc -l |
| 함수 크기 | <= 60줄 | AST / 중괄호 카운팅 |
| 보안 | 크리티컬 0개 | bandit / npm audit + 시크릿 스캔 |

```
└── tests/                         # 실제 커버리지 포함
```

## 세션 복원

세션 중단? 레이트 리밋? 다시 실행하면 됩니다.

```bash
/agent-harmony:harmony
/harmony
# → "태스크 15/23에서 이어하시겠습니까? (a) 이어하기 (b) 처음부터"
```

매 단계마다 상태가 저장됩니다. 아무것도 잃지 않습니다.

## 적합한 프로젝트

| 잘 맞는 프로젝트 | 이유 |
|-----------------|------|
| 웹 앱 (Next.js, FastAPI, Django) | 예측 가능한 구조, 명확한 에이전트 역할 |
| API 서버 | 표준 패턴, 테스트 용이 |
| CLI 도구 | 단순한 구조, 적은 에이전트 필요 |
| SaaS MVP | 인터뷰가 요구사항을 잘 포착 |

| 덜 적합한 프로젝트 | 이유 |
|-------------------|------|
| 대규모 기존 코드베이스 | 암묵적 의존성이 너무 많음 |
| ML/데이터 파이프라인 | 실험적 워크플로우 vs 선형 태스크 |
| 실시간 시스템 | 통합 테스트 자동화가 어려움 |

## 알려진 동작

### 초기화 중 권한 모드 변경

`/project-init` 또는 `/codebase-init` 실행 시 Claude Code가 bypass 모드에서 "accept edits" 모드로 전환될 수 있습니다. 이것은 `.claude/`와 `.mcp.json` 파일에 대한 [플랫폼 레벨 보호](https://code.claude.com/docs/en/permission-modes.md)입니다.

**초기 셋업 시 1회만 발생합니다.** 초기화 완료 후 `Shift+Tab`을 눌러 bypass 모드로 전환하세요.
